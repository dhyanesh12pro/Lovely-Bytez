# Lovely Bitez Website

Premium, mobile-first Next.js website for Lovely Bitez in Kundrathur, Chennai.
The site is built from one central data file so business details, menu items,
links, images, and opening hours stay consistent across pages, SEO metadata,
structured data, sitemap, and owner documentation.

## Tech Stack

- Next.js App Router with TypeScript
- Tailwind CSS
- Framer Motion
- Lucide React icons
- Local images through `next/image`

## Edit Business Data

Update this file first:

```txt
data/site.ts
```

It controls:

- Business name, tagline, phone, address, opening hours, map links
- Instagram, Swiggy, Zomato, WhatsApp visibility
- Page titles, meta descriptions, and canonical paths
- Menu categories and menu cards
- Gallery image paths, alt text, and development credits
- Structured data inputs

Unknown prices use `price: null`, which displays "See current price".
Do not invent prices, reviews, ratings, awards, branch addresses, or delivery
claims.

## Image Replacement

The current files in `public/images` are temporary local copies from the
supplied public references. Before publishing a commercial production website,
replace them with owner-approved original photos using the same filenames or
update `imageAssets` in `data/site.ts`.

Recommended owner photo set:

- Storefront photo
- Indoor seating photo
- Fried chicken close-up
- Burger close-up
- Fries or sides photo
- Dessert photo
- Shake or mojito photo

Use descriptive alt text in `data/site.ts`.

## SEO and Indexing Setup

Generated by the app:

- Unique metadata per page
- Canonical URLs
- Open Graph tags
- Restaurant / FoodEstablishment JSON-LD
- WebSite JSON-LD
- Breadcrumb JSON-LD
- Menu JSON-LD on `/menu`
- `sitemap.xml`
- `robots.txt`
- Web manifest and favicon

Preview and staging deployments are blocked from indexing by default. For the
real production domain, set:

```bash
NEXT_PUBLIC_SITE_URL=https://www.your-real-domain.com
NEXT_PUBLIC_ALLOW_INDEXING=true
NEXT_PUBLIC_INDEXING_DISABLED=false
```

For preview or staging deployments, set:

```bash
NEXT_PUBLIC_INDEXING_DISABLED=true
```

## Deploy to Vercel

1. Push this project to a Git repository.
2. Import the repository in Vercel.
3. Set the production environment variables:

```bash
NEXT_PUBLIC_SITE_URL=https://www.your-real-domain.com
NEXT_PUBLIC_ALLOW_INDEXING=true
NEXT_PUBLIC_INDEXING_DISABLED=false
```

4. Set preview/staging environment variables:

```bash
NEXT_PUBLIC_INDEXING_DISABLED=true
NEXT_PUBLIC_ALLOW_INDEXING=false
```

5. Deploy.
6. Connect the real domain in Vercel.
7. Confirm these URLs work:

```txt
https://www.your-real-domain.com/
https://www.your-real-domain.com/menu
https://www.your-real-domain.com/gallery
https://www.your-real-domain.com/about
https://www.your-real-domain.com/contact
https://www.your-real-domain.com/sitemap.xml
https://www.your-real-domain.com/robots.txt
```

## Owner Actions After Deployment

1. Connect the real domain.
2. Verify Google Search Console and submit the sitemap.
3. Claim and verify Google Business Profile.
4. Keep the same name, address, phone, hours, and website URL everywhere.
5. Upload real shop and food photos regularly.
6. Ask genuine customers for honest Google reviews and reply professionally.
7. Monitor Search Console and update the site, menu, and business information regularly.

## Ethical SEO Notes

This site does not promise number-one Google rankings. It avoids keyword
stuffing, fake reviews, copied content, fake ratings, hidden text, and paid
backlink schemes. Local SEO work should focus on accurate business data,
useful pages, real photos, honest reviews, and consistent updates.
